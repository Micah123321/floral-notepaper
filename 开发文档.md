# 花笺 Floral Notepaper 开发文档

本文档面向花笺项目的开发、维护和交付工作，内容以当前代码库为准。若本文档与代码冲突，以代码为准并同步修正文档。

## 1. 项目概览

花笺 Floral Notepaper 是一个基于 Tauri 2 + React 的本地桌面便签工具。它提供主窗口编辑器、快捷便签小窗、桌面磁贴、Markdown 预览、分类管理、导入导出、多语言、主题和桌面集成能力。

项目当前版本为 `1.0.4`。前端包名为 `floral-notepaper`，Tauri 产品名为 `花笺`，应用 identifier 为 `com.floral-notepaper.app`。

核心目标：

- 本地优先：笔记以 Markdown 文件和 JSON 元数据保存在用户本机。
- 快速调用：托盘和全局快捷键可以快速打开便签。
- 轻量编辑：支持 Markdown、GFM、LaTeX、分栏预览、导入导出和外部文件编辑。
- 桌面贴纸：笔记可以钉为置顶磁贴窗口。
- 跨语言界面：当前包含 `zh-CN`、`zh-HK`、`en-US`。

## 6. 架构总览

花笺采用 Tauri 的典型前后端分工：

```text
React UI
  -> src/features/* 前端 API 与纯函数
  -> @tauri-apps/api/core invoke()
  -> Rust #[tauri::command]
  -> services::notes / desktop
  -> 本地文件系统、窗口、托盘、快捷键、插件
```

前端组件不直接访问文件系统。所有本地文件、配置、窗口和桌面能力通过 Tauri command 或 Tauri 官方前端 API 完成。

## 7. 应用启动流程

### 7.1 前端启动

`src/main.tsx` 执行：

- 获取 DOM 根节点。
- 尝试读取 `getConfig()`，取得 locale。
- 调用 `initializeI18n(locale)`。
- 使用 `I18nextProvider` 挂载 `<App />`。

`src/App.tsx` 执行：

- 通过 `getInitialRoute()` 判断当前窗口视图。
- 读取配置并应用主题、系统主题监听、Tab 缩进和语言。
- 监听 `config-changed` 事件，同步运行时配置。
- 在 Windows 拦截 `Alt+Space`，避免无边框窗口出现系统菜单。
- 按 route 渲染 `MainWindow`、`NotePad` 或 `TileShowcase`。

### 7.2 Tauri 启动

`src-tauri/src/lib.rs` 执行：

- 注册 opener、dialog、single-instance 插件。
- setup 阶段调用 `desktop::setup_desktop()`。
- 注册窗口事件处理。
- 注册所有 `#[tauri::command]`。

`desktop::setup_desktop()` 执行：

- 管理运行时状态和便签窗口池。
- 初始化开机自启和全局快捷键插件。
- 同步开机自启配置。
- 注册全局快捷键。
- 创建托盘。
- 预热便签窗口池。
- 非 `--silent` 启动时显示主窗口。
- 记录冷启动传入的 `.md`、`.markdown` 或 `.txt` 文件路径。

### 8.1.1 提醒预设

主要文件：

- `src/features/reminders/parser.ts`
- `src/components/ReminderInput.tsx`
- `src/features/notes/types.ts`
- `src-tauri/src/services/notes.rs`

数据模型：

- `reminder.kind`：`once`、`weekly`、`monthly`、`workday`
- `reminder.input`：用户输入的原始表达
- `reminder.nextAt`：下一次提醒 ISO 时间
- `reminder.timeOfDay`：本地 `HH:mm`
- `reminder.weekday`：每周提醒时的 1-7 周几
- `reminder.dayOfMonth`：每月提醒时的日期

当前支持的表达包括“明天下午四点”“每周一”“每月五号上午10点”“每个工作日”，以及稍后、今晚、明早、明天下午、下周一、每周一、每月五号、工作日等预设。提醒字段保存在 `metadata.json`，旧元数据缺少 `reminder` 时正常读取为空。

当前 reminder 只表示本地提醒预设和下一次时间，不实现系统通知调度，不接入滴答清单账号或第三方 API。

### 8.3 笔记存储

主要文件：`src-tauri/src/services/notes.rs`

默认数据目录：

- 设置 `FLORAL_NOTEPAPER_DATA_DIR` 时使用该环境变量。
- macOS：`~/Library/Application Support/花笺`
- Windows：`%USERPROFILE%/Documents/花笺`
- 其他环境：当前目录下 `data`

数据文件：

- `config.json`
- `metadata.json`
- `notes/`
- `backgrounds/`
- `attachments/`

重要规则：

- `notes_dir` 保存时会规范化为以 `notes` 结尾。
- 禁止把磁盘根目录或 Windows 系统目录设为笔记目录。
- 删除笔记和分类使用回收站。
- `metadata.json` 损坏时会备份损坏文件并从 Markdown 笔记重建。
- 分类删除会先把笔记移回未分类。
- `metadata.json` 中的笔记条目可包含可选 `reminder`，用于保存本地提醒预设。

### 8.3.1 WebDAV 同步

主要文件：

- `src-tauri/src/services/sync.rs`
- `src/components/WebdavSyncSection.tsx`
- `src/features/settings/api.ts`
- `src/features/settings/types.ts`

同步方式：

- 使用显式上传/下载快照，不做后台自动同步和双向冲突合并。
- 远端文件固定为 `{remotePath}/floral-notepaper-sync.json`。
- 传输使用 WebDAV `MKCOL`、`PUT`、`GET` 和 HTTP Basic Auth。
- 后端会逐级创建远端目录；`remotePath` 会去掉空段、`.` 和 `..`。

快照范围：

- 应用设置中可跨设备共享的字段。
- `metadata.json`，包含分类、笔记元数据、提醒字段。
- `notes/` 下的 Markdown 笔记文件。
- `backgrounds/` 下的应用管理背景图。
- `attachments/` 下的内部笔记附件和附件索引。

安全边界：

- 上传快照不会包含当前设备的 WebDAV 地址、用户名、密码和本机 `notesDir`。
- 上传快照中的 `backgroundImagePath` 只保留文件名，避免泄露本机绝对路径。
- 下载恢复会保留当前设备的 `notesDir` 与 WebDAV 凭据。
- 下载恢复会把远端背景文件名重映射到当前设备的 `backgrounds/` 目录。
- 恢复时先写入 `.sync-restore` 临时区，校验快照路径后再替换受管目录。
- 替换 `notes/`、`backgrounds/`、`attachments/` 前会在 `.sync-backups/` 下保留临时备份；替换失败或元数据写入失败时尝试回滚。

## 12. 开发规范

### 12.1 TypeScript

- 组件负责 UI 状态与事件，领域逻辑优先放入 `src/features/*`。
- Tauri command 调用必须封装在 feature API 中。
- 纯函数应配套 Vitest 测试。
- 用户可见文案优先进入 locale JSON。
- 避免在大型组件继续堆叠独立业务逻辑。

### 12.2 Rust

- 文件系统操作需要先创建父目录。
- 删除用户数据优先使用回收站。
- 路径安全检查必须在目录删除、目录切换和用户选择目录时执行。
- 错误应返回结构化 `AppError`，便于前端本地化。
- 新增配置字段必须提供 serde 默认值，兼容旧配置文件。
- 同步、导出或日志不要写出本机 WebDAV 密码和本机绝对路径，除非该路径是当前设备运行所必需。

### 12.3 UI 与交互

- 当前视觉语言偏纸张、墨色、竹绿色和柔和桌面便签质感。
- 交互状态需覆盖加载、空状态、错误、保存中、已保存、未保存、禁用。
- 无边框窗口的拖拽区域不能吞掉输入、按钮、textarea。
- 触控目标和窗口缩放手柄需要保持可操作。

## 14. 调试建议

前端问题：

- 先用 `npm run dev` 确认 Vite 编译。
- 再用 `npm run tauri dev` 验证桌面 API。
- 关注 `config-changed`、`notes-changed` 等事件是否广播和监听成对出现。

存储问题：

- 临时设置 `FLORAL_NOTEPAPER_DATA_DIR` 指向测试目录，避免污染真实用户数据。
- 检查 `config.json`、`metadata.json` 和 `notes/` 是否一致。
- 元数据异常时验证重建逻辑是否保留所有合法 Markdown 笔记。

窗口问题：

- 检查 label 是否符合 `main`、`notepad-*`、`tile-*`。
- 检查 capability 是否包含目标窗口和所需权限。
- 检查窗口复用、隐藏、关闭和回收路径。

## 15. 安全与数据保护

- 不在仓库中提交真实用户数据、密钥、私钥、系统路径凭据或个人笔记内容。
- Markdown HTML 默认关闭；开启后必须经过 sanitize。
- 外部链接只允许 `http` 和 `https` 通过 opener 打开。
- 用户选择笔记目录时禁止系统根目录和敏感系统目录。
- 删除笔记和分类目录时使用回收站。
- 背景图复制到应用数据目录，替换背景时只删除应用管理目录内的旧背景。
- WebDAV 快照上传前会清空远端快照里的 WebDAV 凭据和本机绝对路径；下载覆盖只作用于当前设备的受管目录。
